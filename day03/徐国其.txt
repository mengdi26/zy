git命令
 配置用户名命令:git config --global user.name "自己用户名";  
 配置邮箱:git config --global user.email "自己邮箱";

验证我那个用户名和邮箱是否已经配置过:
git config user.name
git config user.email

git init		初始化命令
	注意:需要先将本地工作空间中的所有文件提交到本地仓库才能往远程仓库提交
	顺序:	工作空间-->暂存区-->本地仓库-->远程仓库
git add .	将工作空间里面所有的文件和所有文件夹都提交到暂存区
	通过一个命令查看提交的所有文件的状态 
	红色对的代表未提交到暂存区,但是没有提交到本地仓库
	绿色代表已经提交到暂存区了
git status 查看当前工作空间得到状态				
git commit -m "注释"	将暂存区里边的文件提交到本地仓库
git reflog  查看每次提交到本地仓库的版本号(版本号是随机成成的一串字符串)
git reset --hard HEAD^	 回退到上个版本
git reset --hard HEAD^^ 	 回退到上上个版本		..以此类推
git reset --hard 版本号	 回退到某个具体的版本号

注意:多往本地仓库提交,注释要写好(实现一个功能提交一次,修复一个bug提交一次)

关联远程仓库
生成ssh密钥
ssh-keygen -t rsa -C "邮箱名"

本地仓库关联远程仓库的命令
git remote add origin 地址 


把本地仓库的项目提交到远程仓库
git push -u origin master 

要先拉去代码
git pull origin master
git fetch origin master

git pull origin  main --allow-unrelated-histories

如果上面拉去代码的方式不行，只需要创建一个空目录然后通过git clone 地址
远程仓库的链接地址 命令 将远程仓库里面所有的文件 包括readme.md文件都下载到本地工作空间
然后把项目复制到这个目录里面,先提交


创建+切换分支：git checkout –b 分支名


切换分支：git checkout 分支名


查看分支：git branch

创建分支：git branch name


删除分支：git branch –d name



切换到主分支然后合并子分支

合并某分支到当前分支：git merge 子分支
